
**一、** **项目概述**

本项目旨在开发一款专为5-12岁儿童设计的智能硬件产品——儿童AI智能对话打印机。该样机深度融合生成式AI（AIGC）技术与物联网打印技术，通过自然的语音交互，将儿童天马行空的想象力实时转化为可视化的实体画作。它不仅是孩子的"智能玩伴"，更是激发创造力、锻炼语言表达能力的启蒙工具。

**实施策略：两阶段混合推进**

| 阶段                 | 周期             | 目标                       | 投入      |
| ------------------ | -------------- | ------------------------ | ------- |
| **Phase A — 混合验证** | 4周（第1-4周）      | App + 纸上打印模拟，验证数字闭环与实体感知 | 低成本快速验证 |
| **Phase B — 硬件冲刺** | 8周（若Phase A通过） | 完整软硬件一体Demo样机            | 全栈硬件投入  |
|                    |                |                          |         |

**核心决策逻辑：** Phase A验证通过（见"用户测试验收标准"第4周）才进入Phase B。若未通过，重新定位产品方向，不做无效硬件投入。

**二、** **市场背景与痛点分析**

随着AIGC技术的爆发式发展，儿童教育硬件市场正迎来新一轮的智能化升级。多模态大模型（语音识别、文生图）的成熟，使低成本、高智能的实时硬件交互成为可能。

**1. 市场需求**
- 新一代家长重视创造力培养、表达能力及情绪想象力激发
- STEAM教育关注度持续提升，AI玩具/学习机/绘本市场快速增长
- 打印设备正在从"输出工具"转型为"内容创作终端"

**2. 现有痛点**
- 家长希望孩子创造，而不仅是刷视频
- 市面儿童打印机多为"手机照片打印机"或一次性"图片打印机"，缺乏互动性
- 传统故事机功能单一，无法满足"想看看我说的话是什么样子"的探索欲
- 大部分儿童打印机需连接手机App操作，缺乏独立语音互动能力
- 孩子常因"画不像"产生挫败感，缺乏即时正向反馈

**3. 商业假设（已通过用户验证）**
> 家长愿意为"孩子的想象力画作被打印出来"这个价值付费。
>
> 验证依据：已完成的目标用户调研确认此假设成立。
>
> 在Phase A第4周的用户测试中，将进一步量化付费意愿区间和敏感定价点。

**三、** **产品定位及核心功能**

**1. 产品定位**

一款专为5-12岁儿童设计的、通过自然语言对话将想象力转化为实体画作的智能创意陪伴打印机。切入三个高速增长领域的交叉点：AI生成式内容、儿童智能硬件、创意打印与手工内容。差异化核心："儿童通过聊天创造内容，并立即获得实体化结果"。

典型场景：睡前故事创作、亲子互动、DIY涂色、节日创作、情绪表达。

**2. 产品特点**

| 特点       | 说明                 | 验证方式（Phase A）    |

| 虚实结合体验   | 数字AI创造力 → 可触摸实体作品  | 家用打印模拟 + 儿童反馈    |
| 技术验证标杆   | AI内容安全过滤、端云协同交互    | Phase A阶段先验证云端链路 |
| **家长管控** | **所有画作需家长确认后才可打印** | 家长端App核心功能       |

| 功能 | 描述 | Phase A实现 | Phase B补充 |

**3. 数据流与影子路径处理**

主数据流：`[录音] → [ASR] → [Prompt安全审核] → [文生图API] → [图像审核] → [下载] → [打印调用] → [TTS反馈] → [云端存档]`

**影子路径全覆盖：**

| 步骤 | 正常 | 空/nil | 错误 | 超时（15秒） |
|------|------|--------|------|-------------|
| 麦克风录音 | 语音转文字 | "没听到你说什么，再试一次好吗？" | 设备重启提示 | 5秒无音自动结束 |
| ASR识别 | 文字输出 | "能再说大声一点吗？" | 识别结果异常→重试1次 | 超时→"网络好像有点慢" |
| Prompt安全审核 | 通过 | 直接放行 | 命中拒绝→"这个不能画哦，换一个吧" | 正常无需超时 |
| 文生图API（主） | 图片生成 | 描述太抽象→引导更具体的描述 | 内容被拒→替换safe描述重试 | 等待15秒→切备用API |
| 文生图API（备） | 图片生成 | 同上 | 同上→降级为离线缓存图 | 15秒→TTS安抚+重试按钮 |
| 图像审核 | 通过/打回 | 无图→报错告警 | 不适当内容→家长告警 | 5秒超时→人工复核队列 |
| 图片下载 | 本地保存 | 内存不足→清理缓存重试 | 网络断开→"网络断开了，连上后继续" | 超时→断点续传 |
| 热敏打印 | 出纸 | 纸用完→App和机身双重告警 | 卡纸→停机+教程 | 电机故障→TTS报错+自检 |
| TTS反馈 | 语音播报 | 无声→尝试扬声器自检 | 发音异常→静默跳过 | TTS宕机→跳过，不影响打印 |
| 云端存档 | 保存成功 | 空数据→不入库 | 存储满→自动清理30天前 | 超时→本地缓存，联网后同步 |

**4. 多API降级策略**

| 层级 | 提供商 | 角色 | 切换条件 |
|------|--------|------|----------|
| 主API | 通义万相 | 默认使用 | — |
| 第一备 | 文心一格 | 主API超时15秒+2次重试失败 | 自动切换 |
| 第二备 | 腾讯混元 | 前两者均不可用 | 手动切换 |
| 离线降级 | 设备本地缓存图片集 | 所有云端API不可用 | 自动：从10张预设儿童画中随机选一张 |

切换时TTS播报："我正在用另一种方式画，等一小会儿哦"，保证不影响儿童体验。

**六、** **内容安全体系**

**1. 三层过滤架构**

```
第一层：Prompt输入过滤（设备端/App端）
  ├── 敏感词黑名单（内置 + 云端同步更新）
  ├── 意图分类（正常/暴力/色情/自伤/其他）
  └── 年龄适配检查（5-12岁内容分级）

第二层：大模型输出过滤（云端处理层）
  ├── 图像审核API（阿里云内容安全 / 腾讯天御）
  ├── 图像标签分类（人物/动物/场景/武器/危险行为）
  └── 置信度阈值过滤（低于90%安全→进入人工队列）

第三层：家长确认闸门（家长端App）
  ├── 所有画作生成后→推送到家长App预览
  ├── 家长点击"允许打印"→设备方可打印
  └── 家长可开启"自动允许"模式（信任模式）
```

**2. 内容安全响应矩阵**

| 事件 | 对儿童表现 | 对家长表现 | 云端动作 |
|------|-----------|-----------|----------|
| Prompt含敏感词 | "这个不能画哦，换个想法吧" | 推送告警通知 | 记录违规日志 |
| 生成内容被图像审核拦截 | "画好了，但需要爸爸妈妈看一下" | 预览被拦截内容，选择允许/拒绝 | 进入人工审核队列 |
| 重复违规触发阈值（5次/天） | "今天不能画这个哦" | 告警+建议亲子沟通 | 触发人工复核 |
| 家长拒绝打印 | "妈妈说这幅画先不打印哦" | 拒绝原因可备注 | 记录家长偏好 |

**3. 数据隐私合规**
- 儿童语音数据：7天后自动删除录音原文，仅保留ASR文字记录
- 画作数据：默认家长App可见，其他用户不可访问
- 数据存储：中国大陆地区服务器，符合《个人信息保护法》及《儿童个人信息网络保护规定》

**十一、** **每周详细执行计划**

### Phase A — 混合验证（第1-4周）

**第1周：App核心功能开发**
- 开发语音输入界面，实现"按住录音"交互

- 接入文生图API（主：通义万相，开通备用API账号：文心一格）
- 实现文本→图片生成→App内展示的基本闭环
- 搭建第一层内容过滤基础架构（敏感词库+意图分类）
- 开发图片保存到本地功能
- 周复盘：核心链路是否跑通

**第2周：内容安全 + 家长端 + 纸上打印**
- **纸上打印模拟：** 开发打印传输模块，对接家用彩色打印机（或打印店API），实现App→打印机输出
- 完成三层内容安全过滤（Prompt过滤 + 图像审核API接入 + 家长确认闸门）
- 完成第二/第三层内容安全测试（模拟10组攻击性Prompt验证拦截率）
- 家长端App雏形：画作预览列表 + "允许/拒绝打印"按钮
- TTS语音反馈集成（生成等待提示音 + 成功/失败播报）
- 备用API切换逻辑实现（主API超时15秒→自动切备用）
- 整理纸质打印样品，准备用户测试素材

**第3周：体验优化与用户测试准备**
- 优化儿童引导式Prompt（引导5-7岁儿童准确描述）
- 统一AI生成画风（蜡笔/水彩风格）
- 家长端App完善：成长相册 + 分享功能雏形
- 准备10个家庭的测试计划（家长问卷 + 儿童任务卡）
- 打印输出质量评估：家用打印机输出的儿童画作，分类满意度
- API降级策略测试（模拟主API宕机、超时、内容被拒等场景）
- 准备第4周用户测试的测试设备、流程文档

**第4周：用户测试与决策**
- 执行10个家庭用户测试

**用户测试流程：**
```
┌─────────────────────────────────────────┐
│ 1. 家长访谈（15分钟）                   │
│    ├── 背景：孩子年龄/画画习惯/现有玩具  │
│    ├── 痛点验证："想看看我说的话"频率    │
│    └── 付费意愿：¥299/¥399/¥499 敏感度   │
├─────────────────────────────────────────┤
│ 2. 儿童体验（20分钟）                   │
│    ├── 自由探索：孩子自己试用App        │
│    ├── 任务执行：按引导完成3次语音作画   │
│    └── 纸上打印：让孩子拿到打印成品     │
├─────────────────────────────────────────┤
│ 3. 反馈收集（10分钟）                   │
│    ├── 儿童：喜欢吗/还想玩吗/想画什么   │
│    └── 家长：愿意买吗/担心什么/改进建议  │
└─────────────────────────────────────────┘
```

**验收标准：**
- A1通过：≥70%儿童在3次引导内完成有效语音描述
- A4通过：≥60%儿童对打印成品表现出持续兴趣
- A5通过：打印输出满意度≥3.5/5
- A7通过：≥70%家长愿意让孩子独立使用
- 内容安全拦截率：攻击性Prompt的拦截≥95%

- **填写Go/No-Go决策表格，生成Phase A验证报告**

- 完成硬件选型：主控（树莓派/RK3588）、热敏打印模组（58mm/80mm）、麦克风阵列、扬声器、显示屏
- 确认技术栈，开通备用大模型API
- 启动CCC认证咨询，确认认证路径和材料清单

- 物料到货清点，开发板通电测试
- 数据安全评估材料准备
- 外形设计方案完善

**第二阶段：技术原型与POC验证（第7-8周）**

**第7周：打印驱动与图像转换**
- 打印模组驱动开发（走纸/文字打印）

- 对接文生图API，实现图片生成→保存
- 联调：图片生成→自动触发打印
- 修复卡纸、图片错位等基础Bug
- 外形设计最终定稿

**第8周：全链路逻辑串联**
- 物理按键和LED灯接入（按下录音/松开发送）
- 全链路串联：录音→ASR→内容安全→大模型→图像审核→打印→TTS
- 加入TTS语音反馈
- 安全过滤系统从App迁移至设备端
- POC阶段验收：桌面环境成功对话并打印

**第三阶段：结构手板与硬件集成（第9-11周）**

**第9周：结构设计与首版3D打印**

- 供电与布线：内置电池，整理线材

- 功能回归测试：麦克风收音、打印震动稳定性

**第四阶段：系统联调与演示准备（第12-14周）**

**第12周：稳定性测试与体验微调**

- 打印图片对比度与阈值调优
